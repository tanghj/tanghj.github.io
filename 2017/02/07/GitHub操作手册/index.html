GitHub操作手册

  1. 创建文件夹进入终端git init初始化仓库
  2. git add 需要上传的文件
  3. git commit -m “输入备注”
  4. git remote add origin https://github.com/项目地址
  5. git push -u origin master 上传
  • git status 查看文件夹内哪些文件被改动过
  • git diff HEAD —- 文件名 查看文件的改动
  • git log –pretty=oneline 查看提交日志（版本号）
  • git reset —hard HEAD^ 文件夹内所有内容退回上一个版本(HEAD是当前版本)(上一个版本就是“HEAD^”上上一个版本就是“HEAD^^”)
  • git reset –hard 版本号 （版本号没必要写全，前几位就可以了，Git会自动去找。当然也不能只写前一两位，因为Git可能会找到多个版本号，就无法确定是哪一个了。
  • git reflog 查看命令历史，以便确定要回到哪个版本
  • git push origin master 推送最新修改
  • 强制将远程文件覆盖本地文件
    git fetch –all
    git reset –hard origin/master
